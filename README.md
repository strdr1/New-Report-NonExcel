# Fuel Tracking Application

Веб-приложение для учета расхода топлива, реплика Excel-файла "Антон 02.2026.xlsm".

## 📋 Статус проекта

### ✅ Готово

#### Backend (Flask + SQLite)
- ✅ Модели данных: Profile, Year, Month, DailyRecord
- ✅ API endpoints для CRUD операций
- ✅ ValidationService для проверки данных
- ✅ CalculationService для расчетов
- ✅ Поддержка високосных годов
- ✅ Автоматическая инициализация месяцев (12 месяцев при создании года)
- ✅ Сезонные нормы расхода (зима: декабрь-февраль, лето: март-ноябрь)
- ✅ Начальные значения года (спидометр и остаток бензина на конец предыдущего года)

#### Frontend (HTML/CSS/JavaScript)
- ✅ Навигация: Профили → Годы → Месяцы → Ежедневные записи
- ✅ Формы создания/редактирования профилей с fieldsets
- ✅ Форма установки начальных значений года
- ✅ Две таблицы на странице месяца:
  - Таблица ежедневных записей (все дни месяца)
  - Таблица отчета (все дни с объединенными заголовками)
- ✅ Excel-стиль таблиц:
  - Градиентные заголовки (синий)
  - Zebra striping (чередующиеся строки)
  - Желтые ячейки для ввода данных
  - Серые ячейки для рассчитываемых значений
  - Hover эффекты
- ✅ Редактирование ячеек как в Excel:
  - Клик для начала редактирования
  - Enter/Tab для сохранения и перехода
  - Escape для отмены
- ✅ Fill handle (синий квадрат) для копирования значений вниз
- ✅ Контекстное меню (ПКМ) с опциями:
  - Формат ячейки
  - Копировать
  - Вставить
- ✅ Панель форматирования ячеек:
  - Жирный/курсив
  - Размер шрифта
  - Цвет текста
  - Количество десятичных знаков
- ✅ Две кнопки печати:
  - Печать таблицы записей
  - Печать отчёта (с заголовком как в Excel)
- ✅ Пустые значения отображаются как пустые (не 0.0)

#### Формулы и расчеты
- ✅ Спидометр начало = Спидометр конец предыдущего дня
- ✅ Спидометр конец = Спидометр начало + Пробег путевки
- ✅ Бензин остаток = Остаток предыдущего дня - Бензин путевки + Бензин получено
- ✅ Каскадный пересчет при изменении данных

### 🔄 В процессе / Требует доработки

#### Формулы
- ⚠️ **Бензин путевка** - требует уточнения логики:
  - В Excel: `=IF(I48=0, 0, ROUND(I48*$P$4/100, 3))`
  - Где I48 = "За день км" из отчета (не из основной таблицы!)
  - I48 = `IF(D=0 AND C=0, 0, IF(C>0 AND D=0, C-MAX(G), D))`
  - Сейчас реализовано: автоматический расчет при вводе distance_km
  - **Нужно**: проверить соответствие с Excel

#### Вторая таблица (Отчет)
- ⚠️ Структура заголовков - готова (с colspan/rowspan)
- ⚠️ Колонки "Начало дня" и "Конец дня" должны показывать:
  - км (спидометр)
  - л (остаток бензина)
- ⚠️ Расчет "Бензин остаток на конец дня" в отчете

#### Редактирование
- ⚠️ Все ячейки редактируемые (включая рассчитываемые) - реализовано
- ⚠️ Сохранение форматирования ячеек (жирный, цвет, размер) - не сохраняется в БД

### ❌ Не реализовано

## 📚 Референс

### Исходный файл
- **Файл**: `Антон 02.2026.xlsm`
- **Листы**:
  - `Титульный` - данные профиля водителя
  - `Январь`, `Февраль`, ..., `Декабрь` - ежедневные записи и отчеты

### Структура листа месяца в Excel

#### Первая таблица (строки 7-38)
| Колонка | Название | Тип | Формула |
|---------|----------|-----|---------|
| A | Дата | Формула | `=Титульный!A46` (первая), `=A8+1` (остальные) |
| B | День недели | Формула | `=WEEKDAY(A8,1)` |
| C | Спидометр конец дня | Ввод | - |
| D | Пробег путевка | Ввод | - |
| E | Бензин путевка | Формула | `=IF(I48=0,0,ROUND(I48*$P$4/100,3))` |
| F | Спидометр начало | Формула | `=E4` (первая), `=MAX($G$8:G8)` (остальные) |
| G | Спидометр конец | Формула | `=F8+I48` |
| H | Бензин остаток | Формула | `=IF(I49=0,H8,H8-E9+I9)` |
| I | Бензин получено | Ввод | - |

#### Вторая таблица (строки 47-78, отчет)
| Колонка | Название | Формула |
|---------|----------|---------|
| C | Дата | `=A8` |
| D | Начало дня (км) | `=F8` |
| E | Начало дня (л) | Расчет остатка |
| F | Конец дня (км) | `=G8` |
| G | Конец дня (л) | Расчет остатка |
| H | Получено бензина | `=IF(I8=0," ",I8)` |
| I | За день (км) | `=IF(AND(D8=0,C8=0),0,IF(AND(C8>0,D8=0),C8-MAX(G8),D8))` |
| J | За день (л) | `=IF(AND(C8=0,D8=0)," ",E8)` |

### Ключевые ячейки
- `E4` - Спидометр на конец предыдущего года
- `K4` - Остаток бензина на конец предыдущего года
- `P4` - Норма расхода бензина (л/100км)

## 🚀 Установка и запуск

### Требования
- Python 3.8+
- Современный браузер

### Установка
```bash
pip install -r requirements.txt
python init_db.py
```

### Запуск
```bash
# Windows (простой способ)
start_simple.bat

# Или вручную
python run.py
```

Приложение будет доступно:
- Frontend: http://localhost:8080
- Backend API: http://localhost:5000

### Остановка
```bash
stop.bat
```

## 🔧 API Endpoints

### Profiles
- `GET /api/profiles` - Список профилей
- `POST /api/profiles` - Создать профиль
- `GET /api/profiles/{id}` - Получить профиль
- `PUT /api/profiles/{id}` - Обновить профиль
- `DELETE /api/profiles/{id}` - Удалить профиль

### Years
- `GET /api/profiles/{id}/years` - Список годов
- `POST /api/profiles/{id}/years` - Создать год (автоматически создает 12 месяцев)
- `GET /api/profiles/{id}/years/{year}` - Получить год с месяцами
- `PUT /api/profiles/{id}/years/{year}` - Обновить начальные значения года
- `DELETE /api/profiles/{id}/years/{year}` - Удалить год

### Months
- `GET /api/profiles/{id}/years/{year}/months/{month}` - Получить месяц с ежедневными записями
- `GET /api/profiles/{id}/years/{year}/months/{month}/report` - Получить отчет
- `PUT /api/profiles/{id}/years/{year}/months/{month}/days/{day}` - Обновить запись дня

## 📝 Примечания

### Сезоны
- **Зима**: декабрь, январь, февраль (используется fuel_rate_winter)
- **Лето**: март-ноябрь (используется fuel_rate_summer)

### Форматирование чисел
- Спидометр, пробег: целые числа (без десятичных)
- Бензин: 2-3 знака после запятой
- Пустые значения отображаются как пустые ячейки

### Печать
- Отчет печатается с заголовком:
  - "за [месяц] месяц [год] года"
  - "по автомашине [марка]"
  - "Гос. номер [номер]"
  - "Водитель [ФИО]"

## 🐛 Известные проблемы

1. Формула "Бензин путевка" требует уточнения - сейчас рассчитывается от distance_km, но в Excel используется сложная логика с отчетом
2. Форматирование ячеек (цвет, шрифт) не сохраняется между сеансами
3. Вторая таблица (отчет) требует доработки расчета остатков бензина
